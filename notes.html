Notes

  • The SimReady standard as described should only be applied to static props and 3D art assets that don’t move or articulate. Why? While NVIDIA would like to provide a SimReady standard for every simulation feature immediately, that’s not practical. source

Asset Class -> Asset Family -> Asset Category

Materials

“By following specified SimReady material naming conventions , there will be tooling to help convert standard OmniPBR and OmniGlass materials into precise SimPBR physical materials that will generate the correct outputs from SimReady objects so that the RTX sensors can respond properly within the simulation.” source
